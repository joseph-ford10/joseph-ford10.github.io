JavaScript

Let's learn about one of the most important web development technologies

29 January 2023

JavaScript: How does it relate to HTML and CSS?

Along with HTML and CSS, JavaScript is one of the core technologies of the modern web. It is also, perhaps, the most powerful of the three.

You can think of HTML and CSS as specialists with clearly-defined jobs. HTML's job is to present content (like these very words) in a structured way; CSS's job is to take that content and make it pretty.

JavaScript's job is... well, that's the thing. JavaScript doesn't have a single, defined job--it's a generalist. At its most basic, JavaScript can do the same things that HTML and CSS do, presenting content and styling it. It can also, though, build powerful apps within websites. It does all of this by building upon (and changing) the groundwork laid by its specialist siblings.

The DOM

However, when JavaScript adds to or edits a page, it is not making changes to the HTML and CSS files. Instead, it makes changes to a copy of the HTML that is generated when the page loads.

Calling it a ‘copy’ is somewhat inaccurate, actually, because it’s not an exact replica of the HTML. Instead, the page structure is rendered as a hierarchical series of ‘nodes’ that the JavaScript can access and manipulate as necessary. This not-quite-a-copy is called the Document Object Model, or DOM. By manipulating the DOM, JavaScript can make adjustments to the version of the page that a user sees without affecting the underlying HTML file.

This can be useful in all sorts of situations. A classic example would be having some sort of timer or clock on the page. This would not be possible with HTML alone because HTML files are static. JavaScript, however, can put a timer on the page by manipulating the DOM.

So, how does JavaScript interpret the instructions it’s given?

Control flow and loops

Imagine you’ve let your friend borrow your (quite old, quite rundown) car. The next day, your friend phones you asking for help—they can’t seem to get the car to start. As the car’s owner, you know that, because it’s so old and rundown, there’s a very complex, very involved trick to getting it started again when it behaves like this. To pull off this trick, your friend will need to do exactly as you say in exactly the order that you say.

You start issuing instructions down the phone:

  • "Put the key in the ignition"
  • "Wind the window down exactly halfway"
  • "Remove the key from the ignition"
  • "Turn the headlights on"

So far, the number of instructions you’ve given matches the number of times you’ve spoken; one utterance, one instruction. However, you’re coming up to the bit where the steering wheel needs to be tapped exactly six times. You could, if you wanted to, stick to your current rhythm and say:

  • "Tap the steering wheel"
  • "Tap the steering wheel"
  • "Tap the steering wheel"
  • "Tap the steering wheel"
  • "Tap the steering wheel"
  • "Tap the steering wheel"

However, this feels like a waste of breath and energy. Your friend knows how to count, your friend is the one performing the task, so why not just say:

"Tap the steering wheel six times"

Issuing instructions to JavaScript is similar to issuing instructions to your friend. Normally, you will be telling JavaScript to move forward in a linear, top-to-bottom way, one instruction followed by the other. There will be occasions, though, when something will need to be done multiple times. The most efficient way of doing this is take your core instruction (“tap the steering wheel”) and phrase it in a way that makes clear how many times it needs to be done. In the case of JavaScript, this would be done with a loop. For example:

for (let i=0; i < 6; i++) {
alert('tap the steering wheel')
}

As complex as it might look, this really is just a version of the instruction above. The order is slightly different, but even that can be converted into natural language. Instead of saying “tap the steering wheel six times”, the JavaScript instruction is closer to “Okay, you’re going to need to do this next thing six times” (line 1) followed by “tap the steering wheel” (line 2). (Note that JavaScript doesn't have an actual steering wheel to tap here; running this code would simply make the browser pop up six boxes with the phrase "tap the steering wheel" inside.)

Of course, you won’t always know exactly how many times an action will need to be carried out—and that’s fine with JavaScript. Just as you could tell your friend to tap the steering until, say, they hear a certain sound, you could tell a JavaScript loop to run until a particular criterion has been met.

Next, let’s look at arrays and objects.

Arrays and objects

Arrays and objects are, along with variables, important ways of storing data in JavaScript. Let’s start with arrays. Just as a variable can store a single value, arrays can store multiple values in a kind of list. Here’s an example of an array being created and told to store the numbers one to five.

let myArray = [1, 2, 3, 4, 5]

Don’t be fooled by the simplicity of this example, though. Although an array can store a series of single-digit numbers, it can also store many other things—including other arrays! Accessing an item within an array is pretty simple, however. You simply take the name of the array and, using square brackets, specify the index of whatever it is you want to access. If we wanted to pull the number three from the above array, for example, we would simply take the name of the array (myArray) and use the number… two?

myArray[2]

Yes--arrays start counting from zero.

Now let's see an example of an object being declared.

let myObject = {sides:"8", colour:"yellow", size:"500"}

Leaving aside the question of what is being described in this example (I'm not sure either, to be honest), we can see that objects, while bearing some resemblance to arrays, are different. Instead of listing values sequentially, they store values ("8", "yellow", "500", in the example above) in association with keys ("sides", "colour", "size"). These keys can be used to access the values with either square brackets:

myObject[colour]

Or dot notation:

myObject.sides

With arrays and objects covered, let's look at another important JavaScript concept: functions.

What are functions?

Let’s go back to the ‘borrowed car’ situation above. Imagine that part of getting your car to start involves a complex sequence actions that need to be done together. Imagine, also, that this complex sequence of actions needs to be carried out multiple times during the overall sequence of actions that you’re issuing to your friend. One way of handling this would be to issue these instructions each time they came up:

  • "Release the handbrake for one second while tapping the letter F in morse code on the horn"
  • [Different step]
  • [Different step]
  • "Release the handbrake for one second while tapping the letter F in morse code on the horn"

Much like repeating the same instruction six times in a row, though, this would be inefficient. A better solution would be to tell your friend before they start that there are going to be several steps where they’ll need to release the handbrake for one second while tapping the letter F in morse code on the horn. For one thing, this would give them a chance to practice the sequence of actions to make sure they’d got it down. It would also mean that, instead of you having to repeatedly describe the same sequence of actions, you could just refer back to your original description: “do the handbrake morse code thing.”

This is how functions work in JavaScript. Complex sequences of events can be bundled together and called upon at any time by referring to the name. Although JavaScript has certain functions built in, users can also build their own.

function handbrakeMorseCode() {
alert('release handbrake for one second')
 alert('tap letter F in morse code on the horn')
}

Functions can also have parts that are adjustable. In our car analogy, it might be that the third time the sequence of events was carried out, the handbrake needed to be released for two seconds or the morse code needed to be the letter V. With your friend, you could say, “do the handbrake thing for two seconds and the morse code thing with the letter V.” With a function, you can include ‘parameters’ that take a certain value that will be ‘passed’ to the function. The overall code will be the same but it will run with the value that you specify.

function handbrakeMorseCode(brakeTime, codeLetter) {
alert('release' + brakeTime + 'for one second')
 alert('tap letter' + codeLetter + 'in morse code on the horn')
}
